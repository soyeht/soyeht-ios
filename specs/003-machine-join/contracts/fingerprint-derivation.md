# Contract: Anti-phishing fingerprint derivation

Identical implementation required on both repositories (theyos Rust + iSoyehtTerm Swift).

## Input

The candidate's machine public key in 33-byte SEC1 compressed form (`02`/`03` prefix + 32-byte X coordinate).

## Algorithm

```
digest = BLAKE3-256(m_pub_sec1)
bits   = first 66 bits of digest, MSB-first
groups = bits split into six 11-bit groups, MSB-first
words  = for each group, look up index in the standard BIP-39 English word list (2048 entries)
output = words joined with single ASCII space ' ', all lower-case
```

The first 66 bits come from `digest[0..9]` (72 bits) with the last 6 bits ignored. Equivalently:

```
b = digest[0..9]                                      # 9 bytes = 72 bits
i0 = (b[0] << 3) | (b[1] >> 5)                        # bits 0..10
i1 = ((b[1] & 0x1f) << 6) | (b[2] >> 2)               # bits 11..21
i2 = ((b[2] & 0x03) << 9) | (b[3] << 1) | (b[4] >> 7) # bits 22..32
i3 = ((b[4] & 0x7f) << 4) | (b[5] >> 4)               # bits 33..43
i4 = ((b[5] & 0x0f) << 7) | (b[6] >> 1)               # bits 44..54
i5 = ((b[6] & 0x01) << 10) | (b[7] << 2) | (b[8] >> 6)# bits 55..65
```

Each `iN` is in `[0, 2047]` (11 bits) and indexes into the BIP-39 English wordlist.

## Output format

Six lower-case ASCII words separated by single ASCII space (`U+0020`). No trailing whitespace, no punctuation, no Unicode normalization issue (the BIP-39 English list is pure ASCII).

Example: `mango clutch piano fossil bridge zone`

## Determinism contract

For any 33-byte SEC1 input, every implementation MUST produce byte-equivalent output. A test vector table lives at the bottom of this document and MUST be executed by both repositories' test suites.

## Where rendered

- theyos installer console on the candidate machine (Story 1 and Story 2): printed alongside the QR / Bonjour announcement message at install time.
- Soyeht iPhone confirmation UI: shown above an "Approve" / "Decline" button alongside the candidate's hostname.
- theyos owner-events log: stored as the `payload.fingerprint` field of `OwnerEvent{type=join-request}` (verbatim, never re-derived later).

## Test vectors

The canonical cross-repo vectors live at:

```
specs/003-machine-join/tests/fingerprint_vectors.json
```

(repo-root-relative; the spec dir is the agreed anchor between theyos and iSoyehtTerm). Both repositories' test suites MUST consume **this exact path** and assert byte-equality. The file is regenerated by the theyos Rust harness via:

```
cargo test -p household-rs --test fingerprint -- --ignored regenerate_fingerprint_vectors_json
```

Each entry has the following shape:

```json
{
  "index": 0,
  "m_pub_sec1_hex": "02f465b9a16a9e786e4f450980185dc406ec814c72a8b88bf89b74a8516a89391b",
  "fingerprint": "mass museum swamp various model gift",
  "fingerprint_words": ["mass", "museum", "swamp", "various", "model", "gift"]
}
```

Two parallel views of the same data:

- `fingerprint`: the **wire/display form** — six lowercase ASCII words joined by single ASCII space. Used by callers that compare against the human-readable string (`OwnerEvent.payload.fingerprint`, installer console output).
- `fingerprint_words`: the **structured form** — array of six lowercase ASCII strings. Used by callers that prefer to compare per-word (Swift test target's existing convention).

Both forms MUST agree: `fingerprint == fingerprint_words.joined(separator: " ")`. The Rust harness asserts this invariant on every read.

## Why first 66 bits, not first 64

64 bits = 4 groups of 16 (or 5 groups of 12.8 — non-integral). 66 bits is the smallest multiple of 11 ≥ 64, giving exactly 6 BIP-39 words and ~66 bits of comparison material — comfortably above the bit budget needed to defeat any in-window substitution attack while staying readable.
