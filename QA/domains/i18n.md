# Internationalization (15 languages)

## Objective
Verify that every user-facing surface renders correctly in all 15 supported
languages — en (base), pt-BR, pt-PT, es, de, fr, ru, id, ar, ur, ja, hi, mr,
te, bn — without layout breakage, missing keys, or untranslated strings.

## Risk
- A string referenced in code was never added to `.xcstrings` → user sees the
  raw key or English fallback.
- RTL languages (ar, ur) mirror the layout in ways custom AppKit views don't
  anticipate.
- French/German expansion (typically +30% vs English) overflows fixed-width
  buttons in the Welcome cards and Claw Store cards.
- The three catalogs (iOS app, macOS app, SoyehtCore) drift independently —
  coverage tests catch missing translations in any of the three.

## Preconditions
- Latest build of both Soyeht (iOS) and SoyehtMac.
- No real server required — smoke tests cover catalog resolution only.

## Fixtures
- Xcode Scheme → Run → Options → App Language — used to force a runtime locale
  without changing system preferences.
- `TerminalApp/Soyeht/Localizable.xcstrings` — iOS catalog.
- `TerminalApp/SoyehtMac/Localizable.xcstrings` — macOS catalog.
- `Packages/SoyehtCore/Sources/SoyehtCore/Resources/Localizable.xcstrings` —
  shared catalog (APIError, ClawNotificationHelper, UnavailReason, ColorTheme).

## Hardcoded string audit (ST-Q-I18N-010)

Simple `rg 'Text("[a-z]'` has false positives for legitimate technical formatting.
Use the allowlist-aware two-step grep below. Any new `Text(...)` pattern that is a
user-facing English sentence should use `Text("key.name")` (for static text) or
`Text(LocalizedStringResource(key, defaultValue: "...", comment: "..."))` (for
interpolated text). Never use `Text("\(x) some English word")` — the extracted
key won't match what gets added to the catalog (see
`feat/i18n-round2` commit for examples).

```bash
# Step 1: capture all Text("...") calls that are NOT dotted-keys (i.e. possible
# hardcoded English). Catch both lowercase and interpolated starts.
rg 'Text\("[^"]*"\)' TerminalApp/Soyeht TerminalApp/SoyehtMac --include="*.swift" \
  | grep -vE '"\s*[a-z][a-zA-Z0-9]*(\.[a-zA-Z0-9]+)+\s*"' \
  > /tmp/i18n-suspects.txt

# Step 2: drop legitimate exceptions (allowlist of technical/numeric formatting).
grep -vE 'verbatim|i18n-exempt|LocalizedString' /tmp/i18n-suspects.txt \
  | grep -vE '"\s*\d+\s*pt\s*"' \
  | grep -vE '"\[\\\(' \
  | grep -vE '"\\\([^"]*\) / \\\([^"]*\) MB"' \
  | grep -vE '"[-+!]"' \
  > /tmp/i18n-real-violations.txt

# Expected: /tmp/i18n-real-violations.txt is empty.
```

**Allowlist — strings that are intentionally not translated:**
- `Text("8pt")`, `Text("24pt")` — typographic units (FontSizeView.swift)
- `Text("\(bytes) / \(total) MB")` — numeric formatting (ClawCardView.swift)
- `Text("[\(tag)]")` — bracket-wrapped dynamic tags (InstanceListView.swift, MacDetailView.swift)
- `Text("[!] \(error)")` — error-level prefix markers (InstanceListView.swift)
- Single-char symbols: `"+"`, `"-"`, `"/"`
- Anything explicitly marked `// i18n-exempt` or `Text(verbatim:)`

## Known gaps / follow-ups

- **Native-speaker review**: strings for mr, te, bn (+ ur, hi for new plural keys
  added in feat/i18n-round2) were machine-translated and flagged
  `state: "needs_review"`. The coverage test accepts both `translated` and
  `needs_review`. Before a public shipment targeting those locales, route them
  through native reviewers and flip the state to `translated`.
- **RTL layout audit beyond WindowChrome**: feat/i18n-round2 fixed the top-level
  window chrome (WorkspaceTabsView edgeInsets, WindowChromeViewController
  leftInsetGuide). Other custom AppKit containers — QR handoff popover, Welcome
  card internal layouts, Preferences sheet custom controls — have not been
  audited visually in ar/ur. When shipping to RTL markets, schedule a full sweep.
- **Terminal output**: strings that travel from the server (`APIError.http`'s
  server-supplied body, `UnavailReason.installFailed`'s `%@` error) are not
  localized by the iOS / macOS catalogs — they come through as whatever the
  server writes. A future PR should either localize server errors or mark them
  `// i18n-exempt` so the distinction is deliberate.
- **`SoyehtAPIClient.relativeTimeLabel`** (`SoyehtAPIClient.swift:147-153`) —
  still returns "now" / "Xm ago" / "Xh ago" / "Xd ago" hardcoded in English.
  Used in `instancelist.workspace.windowsAndCreated` as `%2$@`. Follow-up PR
  should either (a) use `RelativeDateTimeFormatter` / `Date.formatted(.relative)`
  (already locale-aware), or (b) introduce CLDR plural keys per unit
  (minute/hour/day) plus a "now" literal.
- **Legacy `(s)` strings in existing catalog keys** — e.g.
  `clawSetup.footer.serversAvailable = "%lld server(s) available"`. Lazy English
  that doesn't grammatically work in most non-English languages. Upgrade to
  CLDR plural in a future sweep, using the same shape as the three plural keys
  introduced in feat/i18n-round2.
